## 워크플로우

### Step 1: 목적 자동 감지 (Auto-Routing)

**$ARGUMENTS를 분석하여 목적을 자동으로 판단합니다.**

| 키워드/패턴 | 자동 선택 목적 | 추가 자동 설정 |
|------------|---------------|---------------|
| 이미지, 그림, 사진, 그려줘 | ㅂ. 이미지생성 | 상세도 III, 모델 6 |
| 영상, 동영상, 비디오, 클립 | ㅅ. 동영상생성 | 상세도 III, 모델 6 |
| 코드, 코딩, 개발, 프로그램 | ㄱ. 코딩/개발 | 모델 2 (Claude) |
| 글, 작성, 블로그, 기사 | ㄴ. 글쓰기/창작 | 모델 3 (Gemini) |
| 분석, 데이터, 통계, 비교 | ㄷ. 분석/리서치 | 모델 3 (Gemini) |
| 에이전트, 자동화, 워크플로우 | ㄹ. 에이전트 | 상세도 III, 모델 2 |
| 팩트체크, 사실 확인, 검증 | 팩트체크 | 상세도 III, 모델 1 |
| 조사, 리서치, 찾아줘 | 리서치 | 모델 3 (Gemini) |
| GPT, ChatGPT | - | 모델 1 (GPT-5.2) |
| Claude, 클로드 | - | 모델 2 (Claude) |
| Gemini, 제미나이 | - | 모델 3 (Gemini) |

**자동 감지 동작:**
1. 키워드 매칭 성공 → 목적+모델 자동 선택 + 확인 메시지 출력
2. 추가 옵션 필요 시 → 해당 옵션만 간략히 질문 (전체 폼 생략)
3. 키워드 매칭 실패 또는 모호 → 입력 폼 표시

**확인 메시지 예시:**
```
🎯 자동 감지 결과:
   목적: 코딩/개발
   모델: Claude Opus 4.5 (LMArena 코딩 1위)
   상세도: II (보통)

📐 추가 옵션이 필요하면 알려주세요.
   없으면 바로 프롬프트를 생성합니다.
```

## 프롬프트 생성 워크플로우

### Phase 1: 초안 생성

**CE 체크리스트 (자동 적용)**

| 원칙 | 체크 항목 | 적용 방법 |
|------|----------|----------|
| **U자형 배치** | 중요 정보가 시작/끝에 있는가? | 시작: Critical Rules, Role / 끝: Final Reminder |
| **Lost-in-Middle 방지** | 핵심 규칙이 2회 이상 반복되는가? | 시작 1회 + 끝 1회 명시 |
| **Signal-to-Noise** | 불필요한 서술이 제거되었는가? | 장황한 설명 → 표/글머리로 변환 |
| **Progressive Disclosure** | 단계별로 정보가 구조화되었는가? | 각 Phase에 입력→출력 명시 |

**모델별 필수 블록 확인**

| 모델 | 필수 블록 | 스킬 참조 |
|------|----------|----------|
| GPT-5.2 | `<output_verbosity_spec>` 항상 포함 | gpt-5.2-prompt-enhancement.md |
| Claude Opus 4.5 | 명시적 지시, `<default_to_action>` | claude-4.5-prompt-strategies.md |
| Gemini 3 Pro/Flash | Constraints를 **최상단**에 배치 | gemini-prompt-strategies.md |
| Veo 3.1 | 주제/동작/스타일, 오디오 프롬프트 | gemini-prompt-strategies.md |
| 이미지 모델 | 주제/스타일/분위기, 시그널 강화 | image-prompt-guide.md |

### Phase 2: 전문가 3인 토론 (필수 - 자동 실행)

> **참고**: Multi-Persona Prompting, Panel-of-Experts 기법 적용

**전문가 페르소나 (작업 유형에 따라 동적 생성)**

| 역할 | 기본 전문 분야 | 검토 초점 |
|------|--------------|----------|
| **Expert 1: 프롬프트 아키텍트** | Context Engineering, 토큰 최적화 | 구조, 효율성, 모델 특성 적합성 |
| **Expert 2: 도메인 전문가** | 작업 유형별 전문 지식 (동적 할당) | 내용 정확성, 완전성, 누락 요소 |
| **Expert 3: 심판/통합자** | 비판적 분석, 합의 도출 | 상반된 의견 조율, 최종 결정 |

**동적 도메인 전문가 예시:**
- 코딩 → 시니어 소프트웨어 엔지니어
- 이미지 생성 → 시각 디자이너/아트 디렉터
- 글쓰기 → 편집자/카피라이터
- 팩트체크 → 저널리스트/리서처

**토론 프로세스 (자동 실행)**

```
초안 생성 완료
   ↓
Expert 1 검토: CE 원칙, 모델 블록, 토큰 효율
   ↓
Expert 2 검토: Expert 1 의견 반응 + 도메인 관점 피드백
   ↓
Expert 3 통합: 양측 의견 조율 + 합의점 도출
   ↓
최종 프롬프트 출력
```
